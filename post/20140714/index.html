7/14（火）にNetcom Developer’s Backing #3を行いました。

スマフォアプリ開発に役立つツールやクラウドサービスについて：Webインテグレーション事業部　高柳さん

Webインテグレーション事業部のスマフォアプリ開発チームにおいてどんな環境で開発を行っているかを紹介してもらいました。 また、過去の開発案件や現在進行中のプロジェクトについても紹介していただきました。

モバイル端末の実機検証を効率化するサービス

  1. Scirocco Cloud
    クラウドベースのモバイルテストプラットフォームです。
    ブラウザ経由で端末(実機)のリモート操作やテスト実行ができます。
    Androidとβ版（7/20現在）でiOS、Firefox OS、Ubuntu Touchに対応。 http://www.scirocco-cloud.com/
    高柳さんのチームでは実際にこちらのクラウドサービスを利用しているそうです。

  2. Smartphone Test Farm
    ブラウザからスマホ実機で検証を行うためのシステムです。
    ブラウザ経由で端末(実機)のリモート操作やテスト実行ができます。 サイバーエジェント社内で利用されいたものですが、最近OSS化されました。 Androidのみ対応。
    https://openstf.github.io/
    ※ WBCの社内サーバにも環境が構築されていて試すことができます。
    詳細はWBC小出さんまで。

  3. AWS Device Farm
    7/14に発表されたモバイルアプリのテストをAWSで行うサービスです。
    用意されたAPIを通じて自動テストを行えるとのこと。
    ブラウザからのリモート操作はできないようです。
    AndroidとFire OSに対応。
    http://aws.amazon.com/jp/device-farm/

その他、スマフォアプリ開発時に使用しているもの

  • バージョン管理：Github（プライベートリポジトリ）
  • テスト自動化ツール：Scirocco Cloud
  • CIツール：CircleCI（導入中）
  • アプリ配布ツール：Deploy Gate
  • コミュニケーションツール：Slack、Skype（顧客による）。Qiita Teamも今後使う予定

ちなみにデジタルマーケティング事業部では以下のツールを使っています。

  • バージョン管理：SVN、GitBucket（GitHubクローン）
  • CIツール：Jenkins
  • アプリ配布ツール：？（TestFlightが使われてる場合があるかも、その他はAdHocでビルドして社内サーバにアップしている）
  • コミュニケーションツール：Chat&Messenger（以前はChatWorkなども使われていたが、、）

watchOS-2-Samplerを実機に入れてみた：デジタルマーケティング事業部 高井

iOSのSamplerアプリで有名な堤さんのwatchOS-2-Samplerを入れてみました。
watchOS2で新しく追加されたAPIを使ったサンプルアプリ。 https://github.com/shu223/watchOS-2-Sampler

そのうち、自分でも何か作ってみようと思います。

実機にインストールするときにめんどくさかった点

  1. iOS9betaをペアリングするiPhoneにインストールしないといけない
  2. watchOS2とのペアリング時にiPhone側でメッセージアプリとAppleIDの連携を解除しないといけないことがある
    • メッセージアプリにAppleIDをひも付けている場合にペアリング時のプロンプトでパスワードを入力するように求められるが正しいパスワードをいれても認証に失敗するので、設定 > メッセージをOFFにして連携を解除しました。
  3. WatchKit AppとWatchKit Extensionに固有のbundle identifierを設定しないといけない
    • WatchKit App：iOSアプリのbundle identifier+.watchkitapp
    • WatchKit Extension：iOSアプリのbundle identifier+.watchkitapp.watchkitextension
  4. WatchKit AppとWatchKit Extensionのinfo.plistに親アプリのbundle identifierを設定しないといけない
    • WatchKit App：WKCompanionAppBundleIdentifierにiOSアプリのbundle identifier
    • WatchKit Extension：NSExtension > NSExtensionAttributes > WKAppBundleIdentifierにiOSアプリのbundle identifier+.watchkitapp

余談ですが、高柳さんのチームではこちらのSampler開発者の堤さんが関わるプロジェクトを進行中らしいです。機会があれば、お話を聞いてみたいです。
